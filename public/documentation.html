MyFlix

This is a web application that will provide users with access to information about different movies, genres, and directors. MyFlix users will be able to sign up for access, update their personal information, and create and edit a list of their favorite movies.


Get All Movies

/movies

Method Type: GET

Description: This endpoint allows users to get whole list of movies. Endpoint returns an array with list of objects. Each object contains data about movies.

Query Parameters: None

Expected request data: Required: /movies

Example:

localhost:8080/movies

Expected response data: An array with list of movies. Each object contains data about movies. Every key is described with value of a string.

Example:

{
Title: "Bridesmaids",
Description: "Competition between the maid of honor and a bridesmaid, over who is the bride's best friend, threatens to upend the life of an out-of-work pastry chef.",
Genre: { Name: "Comedy" },
Director: { Name: "Paul Feig"},
ImageURL: "https://m.media-amazon.com/images/I/71NSaiNKO9L._AC_SL1333_.jpg"
}

Search Movies by Title

/movies/:Title

Method Type: GET

Description: This endpoint allows users to search a movie by title. Endpoint returns an object with data about specific movie.

Query Parameters: :movieTitle

Expected request data: Required: :movieTitle

Example:

localhost:8080/movies/The%20Notebook

Expected response data: An object containing data about specific movie. Every key is described with value of a string.

Example:

{
Title: "The Notebook",
Description: "A poor yet passionate young man falls in love with a rich young woman, giving her a sense of freedom, but they are soon separated because of their social differences.",
Genre: { Name: "Drama" },
Director: { Name: "Nick Cassavetes" },
ImageURL: "https://www.imdb.com/title/tt0332280/mediaviewer/rm1153669376/?ref_=tt_ov_i" }

Response status codes:

200: OK- JSON object

400: Bad Request- No such movie!

Adding Users

/users

Method Type: POST

Description:This endpoint allows users to register. Endpoint returns a JSON object with few key values. For key "name" it returns string,for "favorite movies" array, and for an "id" string.

Query Parameters: None

Expected request data: JSON object with data about user. Required: "Username"

Example:

{
"Username": "brielle715"
}

Expected response data: JSON object with data about user which includes generated ID

Example:

{
"Username": "brielle715",
"favoriteMovies": [],
"id": "1"
}

Response status codes:

201: Created

400: Bad request- Username already exists!


Updating Usernames

/users/:Username

Method Type: POST

Description: This endpoint allows users to update their username. Endpoint returns a JSON object with few key values. For key "name" it returns string, for "favorite movies" array, and for an "id" integer.

Query Parameters: :Username

Expected request data: JSON object with data about user.

Required: :Username

Example:

localhost:8080/users/brielle715


{
"Username": "newUsername"
}

Expected response data: JSON object with data about user which includes,"id" with integer value, "Username", and "favorite movies" with an array.

Example:

{ "id": 1,
"Username": "Brielle Updated Name",
"favoriteMovies": [] }

Response status codes:

200: OK-

400: Bad request- No such user!


Add Favorite Movies

/users/:Username/movies/:MovieID

Method Type: POST

Description:This endpoint allows users to add movie to their favorite list.Endpoint returns an JSON object with users data including updated favorite movies array.

Query Parameters: :Username, :MovieID

Expected request data: Required: :Username, :MovieID

Example:

localhost:8080/users/brielle715/631a0654a7fa1a84728fb357

Expected response data: JSON object with updated list of movies in Favorite Movies array.

{ 631a0654a7fa1a84728fb357 has been added! }

Response status codes:

200: OK- Movie has been added to user array!

400: Bad Request- No such user!


Remove Movie from Favorite

/users/:Username/movies/:MovieID

Method Type: DELETE

Description: This endpoint allows users to remove a movie from their favorites. Endpoint returns an JSON object with users data including updated favorite movies array.

Query Parameters: :Username, :MovieID

Expected request data: Required: /:Username, /:MovieID

Example:

localhost:8080/users/brielle715/movies/631a0654a7fa1a84728fb357

Expected response data: A message indicating the movie has been removed from the user's array.

Example:

{ 631a0654a7fa1a84728fb357 was deleted! }

Response status codes:

200: OK- Movie has been removed from users array!

400: Bad Request- No such user!
